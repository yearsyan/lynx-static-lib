# Local Lynx Patches

This directory contains compatibility patches applied on top of the pinned Lynx
submodule by `scripts/sync_lynx_deps.py`.

We keep these changes as patches instead of committing them inside
`third_party/lynx` because Lynx is an upstream-owned submodule. The main
`lynxlib` repository should pin a Lynx commit and carry only the small local
deltas needed to build and run the Windows static SDK.

## `0005-desktop-core-js-loader-returns-source.patch`

This patch makes the Windows/Desktop `CoreJsLoaderManager` hook suitable for a
static SDK that embeds `lynx_core.js` directly into `lynx_static.lib`.

Upstream desktop code currently treats `ICoreJsLoader::GetCoreJs()` as a file
path and then reads that path from disk. That works for Explorer, which copies
`lynx_core.js` next to the executable, but it prevents a static library package
from supplying the runtime core script as compiled-in bytes. The local SDK
compiles a small loader object into `lynx_static.lib`; that loader returns the
embedded script source from a generated byte-array header. The patch changes
the desktop loader to consume that returned source directly when the hook is
registered, while preserving the existing file-based fallback when no loader is
installed.
